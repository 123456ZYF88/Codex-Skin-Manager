# Codex Skin Manager 资料库体验升级设计

日期：2026-07-20  
状态：已确认，待实施计划

## 背景

Codex Skin Manager 已具备主题浏览、导入、切换、最近使用、恢复原版和菜单栏快捷操作，但当前主窗口以卡片画廊为中心，主题选择、当前生效状态、引擎状态和操作反馈集中在同一层级。主题数量增加后，用户难以快速比较主题，也缺少导出、搜索和明确的应用前后验证流程。

本次设计参考以下 MIT 开源项目公开呈现的交互模式：

- Codex NN 的状态首页、运行控制和主题资料库分区；
- LouisDM/CodexSkinManager 的列表选择、大图详情和选择后确认；
- Wangnov/Codex-App-Manager 的拖拽导入、试穿语义和清晰状态反馈。

本项目只借鉴通用信息架构和交互模式，不复制第三方品牌、图标、图片、文字、CSS 或专有视觉资产。应用继续使用现有寒龙、冰晶和武器轮廓视觉语言。

## 目标

1. 让用户一眼区分“当前选中的主题”和“Codex 已验证生效的主题”。
2. 在主题数量增加后仍能通过搜索、筛选和排序快速定位主题。
3. 提供大图详情预览，避免用户必须在多张小卡片之间比较。
4. 支持安全拖拽导入与主题导出，同时维持现有 `.codexskin` 安全边界。
5. 让应用、重试、恢复和引擎状态具有明确、可验证的反馈。
6. 保持原生 SwiftUI、现有 Dream Skin 引擎桥接方式和菜单栏能力。

## 非目标

以下能力不属于本期：

- 在线主题市场和任何远程下载；
- Wangnov 多素材 `.codexskin` 包格式；
- AI 主题设计器或 MCP 主题制作服务；
- 应用自动更新；
- 删除已安装主题；
- 修改 Codex 官方应用、`app.asar`、签名、账号或模型配置。

## 产品结构

### 左侧导航

主窗口保留 `NavigationSplitView`，左侧包含三个一级入口：

- `首页`：当前主题、引擎状态和日常操作；
- `主题库`：全部主题、搜索、筛选和详情；
- `最近使用`：按最近应用时间显示主题，复用主题库详情体验。

侧栏底部固定显示：

- Codex 是否运行；
- Dream Skin 注入器是否存活；
- CDP 是否通过验证；
- 当前已验证主题名称。

状态使用文字、图标和颜色共同表达，不只依赖颜色。

### 首页

首页使用“当前主题主视觉 + 运行控制”结构：

- 左侧或主区域显示当前主题 16:9 大图、主题名称和外观模式；
- 右侧显示 Codex、注入器、CDP 和主题验证状态；
- 提供 `应用当前主题`、`重新启动 Codex`、`暂停主题`、`恢复原版` 操作；
- 不存在活动主题时显示原版 Codex 状态和进入主题库的主要按钮。

`恢复原版` 始终使用破坏性确认对话框，不与普通主题操作混排成同等优先级。

### 主题库

主题库采用“紧凑列表或画廊 + 大图详情”结构。

宽窗口中：

- 左侧内容列为主题列表，显示缩略图、名称、外观、最近使用和活动标识；
- 右侧详情显示大图、名称、主题 ID、版本或来源信息、外观模式和操作区；
- 点击列表只改变选中项，不触发主题应用。

窄窗口中：

- 主题列表变为自适应画廊；
- 选择主题后进入详情，或由可折叠详情区域展示；
- 不保留空白详情列。

窗口最小目标尺寸为 `960×640`。在 `1024×700`、`1440×900` 和更宽窗口下均应保持信息完整。

### 顶部工具栏

顶部工具栏集中提供：

- 导入主题；
- 导出所选主题；
- 刷新资料库和引擎状态。

恢复原版位于首页运行控制和应用菜单中，不作为主题资料库的普通工具按钮。

### 操作反馈

窗口底部使用固定 `OperationBanner` 展示：

- 当前操作阶段；
- 成功或失败结果；
- 必要时的重试和复制错误详情；
- 运行中进度指示。

Banner 不覆盖主题列表或详情按钮。成功消息可自动弱化，失败消息在用户处理前保持可见。

## 视觉规范

### 色彩

- 深海蓝黑用于窗口背景和次级表面；
- 冰蓝只用于主要操作、选中轮廓和活动主题；
- 翡翠绿只表示已验证成功；
- 橙色用于等待或需要重启；
- 红色仅用于失败和恢复原版等危险动作。

颜色必须配合图标与文字状态，避免色觉障碍用户无法判断。

### 形状与层次

- 武器切角保留在选中主题、详情面板和主要按钮；
- 普通列表项采用更克制的圆角或轻微切角，不让所有卡片同时发光；
- 主题图片是主要视觉焦点，预览图不叠加过重渐变；
- `MangaBurstBackground` 降低对比度和线条密度，避免干扰内容。

### 字体

- 页面标题和主题名称可继续使用系统衬线设计，保留漫画与武库气质；
- 正文、状态、筛选和按钮使用系统无衬线字体；
- ID、版本和引擎信息使用等宽字体；
- 不引入未授权或需要额外分发许可的字体文件。

### 动效

仅保留：

- 页面内容淡入；
- 选中主题切换；
- 操作状态变化；
- 详情展开与收起。

启用系统“减少动态效果”时关闭非必要动画。动画不得改变控件最终布局或阻止操作。

## 搜索、筛选和排序

搜索范围：

- 主题名称；
- 主题 ID。

筛选项：

- 全部；
- 深色；
- 明亮；
- 自动；
- 最近使用。

排序项：

- 最近使用优先；
- 名称升序。

无结果时区分“资料库为空”和“当前筛选无匹配”，并提供清除筛选或导入主题操作。

## 导入流程

入口：

- 工具栏 `导入主题`；
- 将 `.codexskin` 文件拖入主题库或主窗口。

阶段：

1. 检查文件扩展名和安全作用域；
2. 验证 ZIP、文件数量、路径、大小和图片类型；
3. 验证 `theme.json` 与图片引用；
4. 检查主题 ID 是否重复；
5. 原子安装到主题库；
6. 刷新资料库并选中新主题。

导入成功后不自动应用主题。

重复主题 ID 时显示现有主题与待导入主题的名称和 ID。只有用户明确选择 `替换已安装主题` 才允许覆盖。取消操作不改变现有主题。

## 导出流程

只有选中有效主题时允许导出。导出的 `.codexskin` 继续采用安全精简格式：

```text
theme.json
background.png  # 也可以是受支持的 JPEG 或 WebP 文件
```

导出要求：

- 包内恰好两个平铺文件；
- 不包含绝对路径、用户目录、日志、最近使用记录、引擎状态或临时文件；
- 不包含脚本、CSS、二进制、符号链接或远程资源；
- 导出后使用同一套导入校验器重新验证；
- 文件名默认由安全化后的主题名称或 ID 生成，用户可在保存面板修改。

导出不代表素材自动拥有公开再分发授权。应用在导出完成后提示用户自行确认主题图片权利，但不读取或上传任何文件。

## 应用主题流程

1. 用户在主题库选择主题；
2. 详情区域展示预览和元数据；
3. 用户点击 `装备主题` 或使用快捷键；
4. 应用检查 Codex、注入器和 CDP 状态；
5. 可热切换时直接切换；必须重启时先说明原因并请求确认；
6. 引擎执行切换；
7. 应用重新读取状态；
8. 只有返回目标主题 ID 且 CDP、注入器均正常时，才标记为 `已启用`。

失败时保持当前成功主题的展示状态，不把选中主题误标为活动主题。用户可重试或复制经过长度限制的错误摘要。

## 状态模型

`AppModel` 继续作为唯一 UI 状态源，管理：

- 当前选中导航区；
- 主题资料库和最近使用；
- 当前选中主题 ID；
- 当前已验证活动主题 ID；
- 搜索、筛选和排序；
- 引擎状态；
- 全局操作状态；
- 待确认的重复导入或重启操作。

视图不得自行推断活动主题，也不得绕过 `AppModel` 直接执行引擎脚本。

## 组件边界

### `DashboardView`

展示当前主题、引擎状态和日常控制，只消费 `AppModel` 提供的状态和动作。

### `ThemeLibraryView`

负责搜索、筛选、排序、列表或画廊布局，以及更新选中主题。它不直接应用主题。

### `ThemeDetailView`

展示所选主题的大图、元数据和应用操作。未选中主题时提供明确空状态。

### `ThemeToolbar`

承载导入、导出和刷新入口，并根据操作状态和选中主题控制可用性。

### `OperationBanner`

统一展示进度、成功、失败、重试和错误复制操作。

### `ThemePackageExporter`

只负责把已验证主题写成安全精简包。它不切换主题、不修改资料库、不读取用户账号数据，并通过可替换接口接受文件系统与归档实现，便于单元测试。

### 现有组件

- `ThemeCatalog` 继续负责读取本地主题和活动主题；
- `EngineBridge` 继续负责调用受控脚本和解析结果；
- `ProcessRunner` 继续保证参数不经 Shell 插值并限制输出；
- `MenuBarContentView` 复用同一 `AppModel`，保持最近主题、导入和恢复状态一致。

## 错误处理

- 文件选择取消不显示错误；
- 无效主题包给出可理解的原因，不展示内部临时路径；
- 重复主题通过确认流程处理，不归类为普通失败；
- 应用失败保留上一次成功状态，并允许重试；
- 引擎状态未知时仍允许刷新、恢复和打开主窗口；
- 错误详情限制长度，复制内容不得包含主题图片数据、环境变量或凭据；
- 恢复原版失败时明确提示 Codex 当前状态，并保留再次恢复入口。

## 快捷键和辅助功能

快捷键：

- `⌘O`：导入主题；
- `⇧⌘E`：导出所选主题；
- `⌘↩`：应用所选主题；
- `⇧⌘R`：恢复原版；
- `⌘F`：聚焦搜索框。

辅助功能要求：

- 所有图标按钮提供明确的 accessibility label；
- 主题列表支持完整键盘导航和可见焦点；
- 活动、选中、失败和等待状态都有非颜色表达；
- 图片预览使用主题名称作为辅助说明；
- 动态状态变化通过适度的辅助功能通知反馈；
- 文字和控件在系统放大字体下不截断关键操作。

## 测试策略

### 单元测试

- 搜索主题名称和 ID；
- 外观筛选与最近使用筛选；
- 名称和最近使用排序；
- 选中主题不改变活动主题；
- 应用成功后仅在验证目标 ID 时更新活动状态；
- 应用失败保留原活动主题；
- 导出包只有两个安全文件；
- 导出包不包含绝对路径或隐私状态；
- 导出后可通过导入校验；
- 重复导入替换和取消；
- 操作繁忙时拒绝第二个冲突动作。

### UI 编译和结构测试

- 新视图与快捷键可编译；
- 首页、资料库、详情和 Banner 均由主窗口引用；
- 导入支持文件选择和拖放；
- 恢复原版继续包含确认流程；
- 菜单栏继续引用同一模型。

### 视觉验收

- `1024×700`、`1440×900` 和宽屏窗口无重叠、截断或无效空白；
- 宽屏列表详情和窄屏画廊均可操作；
- 深色背景上文字和按钮具有足够对比；
- 系统减少动态效果生效；
- VoiceOver 与键盘导航可完成导入、选择、应用和恢复。

### 发布验证

- 现有测试全部通过；
- 新增资料库、详情、导出和快捷键测试通过；
- 构建、安装和启动 `.app`；
- `codesign --verify --deep --strict` 通过；
- 应用图标、版本和 Finder 显示保持正常；
- 在临时主题库中完成一次导出后重新导入的闭环验证。

## 验收条件

设计实现完成需同时满足：

1. 用户能从首页确认当前主题和引擎状态；
2. 用户能搜索、筛选、排序并预览主题；
3. 选择主题不会意外应用；
4. 应用完成后必须经过引擎状态复核；
5. 用户能通过按钮或拖拽安全导入；
6. 用户能导出并重新导入所选主题；
7. 失败、重试和恢复流程可理解且可逆；
8. 所有自动测试、安装验证、签名验证和视觉验收通过；
9. 不引入网络访问、第三方视觉资产或新的隐私数据收集。
