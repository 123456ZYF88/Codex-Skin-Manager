# Codex Skin Manager 应用图标设计

## 目标

为 macOS 应用提供可在 Finder、Dock、应用切换器和“应用程序”目录中正确显示的原生图标，替换当前系统默认占位图标。图标延续应用已经确定的漫画武器视觉，同时避免使用敖丙或其他受版权保护的角色形象。

## 视觉设计

- 画布：`1024 × 1024`，深海军蓝背景，保留约 10% 的安全边距。
- 主徽章：冰晶盾牌与青蓝长枪组成的对称武器纹章。
- 辅助元素：少量冰裂纹、冷色辉光和青色能量线；不使用文字和人物。
- 形态：符合 macOS 图标的方形圆角视觉，缩小到 `16 × 16` 时仍能识别盾牌和枪尖轮廓。
- 素材：使用原创生成图，不引入第三方角色、商标或仓库外素材。

## 构建接入

1. 保存一份 `1024 × 1024` 原始 PNG 到 `Resources/AppIcon-1024.png`。
2. 生成包含 Retina 尺寸的 `Resources/AppIcon.icns`。
3. 在 `Info.plist` 中声明 `CFBundleIconFile` 为 `AppIcon`。
4. 构建脚本把 `AppIcon.icns` 复制到应用包的 `Contents/Resources/`。
5. 将内部构建号递增，促使 LaunchServices 和 Dock 刷新同一 Bundle ID 的图标缓存。

## 失败处理

- 原始 PNG 尺寸、格式或生成的 `.icns` 不合法时，构建直接失败，不产出缺图标的应用包。
- 构建完成后必须验证 Info.plist 声明、资源文件存在性和代码签名。
- 如果 Dock 仍使用旧缓存，先退出旧进程并重新打开新构建；必要时移除并重新固定 Dock 图标，不修改系统安全设置。

## 测试与验收

- 回归测试在修复前因缺少 `CFBundleIconFile` 或图标资源而失败。
- 构建产物必须包含 `Contents/Resources/AppIcon.icns`，且 `Info.plist` 能解析到该资源。
- `iconutil`/`sips` 能读取生成图标，应用代码签名验证通过。
- 实机重启应用后，Finder/Dock 不再显示 macOS 默认网格占位图标。
- 现有主题切换、导入、恢复和侧栏测试全部保持通过。

## 非目标

- 不修改 Codex/ChatGPT 官方应用图标。
- 不加入在线图标下载、用户自定义应用图标或多套图标切换功能。
