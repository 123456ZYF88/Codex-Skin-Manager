# Codex Skin Manager App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an original ice-weapon macOS application icon that renders correctly in Finder, Dock, and the application switcher.

**Architecture:** Keep one original 1024px PNG and one generated ICNS in `Resources/`. The existing shell bundler copies the ICNS into `Contents/Resources`, while `Info.plist` declares the icon by resource name. A source-level packaging test prevents future builds from silently dropping either the declaration or resource.

**Tech Stack:** Swift 6 test harness, macOS `sips` and `iconutil`, Bash bundler, property-list metadata, image generation.

## Global Constraints

- The icon is original artwork: deep navy background, ice-crystal shield, cyan spear crest, no text, people, copyrighted characters, or third-party marks.
- The source image is exactly `1024 × 1024` PNG with about 10% visual safe margin.
- Support macOS 13 and later without new runtime dependencies.
- The application bundle remains ad-hoc signed with identifier `dev.codexskin.manager`.
- Do not modify the official Codex/ChatGPT application or its icon.

**Files:**

- Test: `Tests/CodexSkinManagerTests/UICompileTests.swift`

**Interfaces:**
- Consumes: existing `expect(_:_:file:line:)` test helper.
- Produces: `UICompileTests.appBundleDeclaresIcon()` packaging regression check.

- [ ] **Step 1: Add the failing test**

Add this call at the beginning of `UICompileTests.run()`:

```swift
try appBundleDeclaresIcon()
```

Add this method to `UICompileTests`:

```swift
private static func appBundleDeclaresIcon() throws {
    let testFile = URL(fileURLWithPath: #filePath)
    let projectRoot = testFile
        .deletingLastPathComponent()
        .deletingLastPathComponent()
        .deletingLastPathComponent()
    let infoURL = projectRoot.appendingPathComponent("Resources/Info.plist")
    let plist = try PropertyListSerialization.propertyList(
        from: Data(contentsOf: infoURL),
        options: [],
        format: nil
    ) as? [String: Any]

    try expect(plist?["CFBundleIconFile"] as? String == "AppIcon", "Info.plist must declare AppIcon")
    try expect(
        FileManager.default.fileExists(
            atPath: projectRoot.appendingPathComponent("Resources/AppIcon-1024.png").path
        ),
        "The 1024px icon source must be committed"
    )
    try expect(
        FileManager.default.fileExists(
            atPath: projectRoot.appendingPathComponent("Resources/AppIcon.icns").path
        ),
        "The generated ICNS resource must be committed"
    )

    let buildScript = try String(
        contentsOf: projectRoot.appendingPathComponent("Scripts/build-app.sh"),
        encoding: .utf8
    )
    try expect(
        buildScript.contains("Resources/AppIcon.icns"),
        "The app bundler must copy AppIcon.icns"
    )
}
```

- [ ] **Step 2: Run the test and verify RED**

Run: `/bin/bash Scripts/test.sh`

Expected: exit `1` with `FAIL: Info.plist must declare AppIcon`.

### Task 3: Declare and bundle the icon

**Files:**
- Modify: `Resources/Info.plist`
- Modify: `Scripts/build-app.sh`
- Test: `Tests/CodexSkinManagerTests/UICompileTests.swift`

**Interfaces:**
- Consumes: `Resources/AppIcon.icns` from Task 2.
- Produces: an application bundle whose metadata resolves `AppIcon.icns` from `Contents/Resources`.

- [ ] **Step 1: Add the plist declaration and bump the build**

Add:

```xml
<key>CFBundleIconFile</key>
<string>AppIcon</string>
```

Set `CFBundleShortVersionString` to `1.0.1` and `CFBundleVersion` to `2` so LaunchServices refreshes the same Bundle ID.

- [ ] **Step 2: Copy the ICNS into the app bundle**

```bash

/bin/chmod 644 "$TEMP_APP/Contents/Resources/AppIcon.icns"
```

- [ ] **Step 3: Run the test and verify GREEN**

Run: `/bin/bash Scripts/test.sh`

Expected: `PASS: UICompileTests` and `PASS: all Codex Skin Manager tests`.
